Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
<nil>
false
```
```
Программа выводит две строки. В первой строке распечатано значение переменной err равное nil. Переменная err получает
от функции Foo() интерфейс error, в котором содержится указатель на объявленную структуру по алиасу os.PathError.
Внутри Foo() объявлена переменная err с типом указатель на структуру os.PathError. Далее значение err возвращается
функции, происходит приведение к интерфейсу error и полученный результат присваивается в переменную err функции main().
Так как в интерфейс error передана только декларация структуры без её инициализации, то fmt.Println вывела нулевое
значение структуры равное nil.

Во второй строке выводится результат сравнения переменной err с типом nil равный false. Так как интерфейс error в
переменной err содержит информацию о переданном в него типе отличном от nil, то он при сравнении всегда будет
не равным nil. Только error(nil) будет равно nil при сравнении, но не error(*MyError).

Интерфейс внутри представляет собой структуру iface, содержащую свойства tab с типом указатель на алиас itab
структуры abi.ITab и data с типом unsafe.Pointer.
Пустой интерфейс внутри представляет собой структуру EmptyInterface, содержащую свойства Type с типом указатель на
структуру abi.Type и data с типом unsafe.Pointer.
Интерфейс отличается от пустого интерфейса наличием методов для реализации и наличием информации о метаданных типа
интерфейса InterfaceType.
```
